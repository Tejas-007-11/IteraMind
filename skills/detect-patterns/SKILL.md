# Detect Patterns

## Instructions

1. Analyze relationships between:
   - parameters and performance
   - feature usage and outcomes

2. Identify patterns such as:
   - consistent improvements with certain parameters
   - diminishing returns
   - unstable configurations

3. Detect anomalies:
   - unexpected spikes or drops
   - inconsistent results

4. Output:
   - key patterns discovered
   - correlations between variables
   - anomalies and possible explanations
